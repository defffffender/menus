# Menus — онлайн-меню и заказы для общепита (Узбекистан)

SaaS-платформа для ресторанов, кафе и чайхан: заведение регистрируется, ведёт
онлайн-меню, генерирует QR на столы и принимает заказы прямо в кабинет.
Гость **не регистрируется** — сканирует QR, смотрит меню, заказывает; заказ
привязан к конкретному столу. Всё трёхъязычное: **ru / uz / en**.

Подробный план и статус — в [PLAN.md](PLAN.md).

## Стек

- **Backend:** Django 5.2 (Python 3.12)
- **Frontend:** Django-шаблоны + HTMX + Alpine.js + Tailwind (CDN)
- **БД:** SQLite (dev) → PostgreSQL (prod)
- **Картинки:** Pillow · **QR:** `qrcode` · **SMS:** Eskiz.uz (на старте — заглушка)
- **Вход:** passwordless по телефону +998 (SMS-код)

## Возможности (готово)

- Регистрация заведения и вход по SMS-коду
- Кабинет с тёмным сайдбаром, переключатель языка ru/uz/en
- Столы + генерация и печать QR-кодов
- Меню: категории, блюда (фото, атрибуты, варианты/размеры), стоп-лист
- Публичное меню гостя по QR + корзина (Alpine)
- Заказы: приём от гостя, доска статусов в кабинете (HTMX-опрос),
  экран статуса для гостя
- Роли и права (RBAC): владелец, директор, управляющий, менеджер, официант, кухня

## Структура

```
apps/core        — лендинг, i18n интерфейса
apps/accounts    — пользователи, вход по телефону (SMS-OTP)
apps/restaurants — заведение (тенант), сотрудники, столы, QR, права
apps/menu        — категории, блюда, варианты
apps/orders      — заказы, позиции, статусы
```

## Запуск (dev)

```bash
python -m venv venv
venv\Scripts\activate          # Windows
pip install django pillow qrcode
python manage.py migrate
python manage.py createsuperuser   # телефон вместо username
python manage.py runserver
```

Загружаемые медиа лежат в `media/` (в git не версионируются).

> ⚠️ Перед продакшеном (Этап 7): вынести `SECRET_KEY` и `DEBUG` в окружение,
> подключить реальный Eskiz SMS, переехать на PostgreSQL, собрать Tailwind.
